Networking & Command Reception

The malware establishes a raw socket using socket(2, 3, 0x11). The parameters correspond to PF_INET (2), SOCK_RAW (3), and IPPROTO_UDP (17). This allows the binary to receive UDP packets and inspect their headers directly. It enters a loop calling recv to listen for incoming commands.

Command Execution

When a packet is received, the malware parses the payload. It verifies a specific ID key (idkey) to authenticate the command. If valid, it decrypts the command using myDecryptFun (XOR cipher) and passes it to remoteExec. This function uses popen to execute the shell command and reads the standard output into a buffer.

C2 Communication

Results are sent back using sendResult2Peer. This function constructs a UDP packet, encrypts the result payload with myEncryptFun, calculates the checksum using udp_tcp_checksum, and transmits the data.
