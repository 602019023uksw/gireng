1. Executive Summary

This binary is a Linux x86 malware written in C++ that functions as a sophisticated backdoor equipped with SOCKS proxy and port forwarding capabilities. Upon execution, it initializes a complex runtime environment utilizing C++ standard library components, specifically std::thread for concurrency and std::shared_ptr for memory management. The malware establishes a Command and Control (C2) connection via a cclient component and uses strong encryption (RSA, AES, HMAC) to secure communications and obfuscate its configuration. Its primary objectives appear to be turning the infected host into a proxy node or a pivot point for network attacks, leveraging cmy_socks_mgr (SOCKS manager) and cport_fwd_mgr (Port Forwarding manager). The threat level is High due to its networking capabilities, encryption usage, and potential for integration into larger botnet operations.

2. Malware Capabilities

Identified capabilities and behaviors exhibited by this malware sample.
  • SOCKS Proxy: Implements a full SOCKS proxy server via the cmy_socks_mgr class to route attacker traffic through the infected host.
  • Port Forwarding: Uses the cport_fwd_mgr and multi_fwd classes to manage TCP/UDP port forwarding rules, enabling network tunneling.
  • C2 Communication: Connects to remote controllers using the cclient class with multi-threaded handling.
  • Cryptographic Operations: Supports RSA (Public/Private key), AES (128/192/256-CBC), and HMAC (SHA-1/224/256) for encrypting C2 traffic and configuration data.
  • System Reconnaissance: Gathers system information by reading /proc filesystem entries (cpuinfo, mounts, net).
  • Persistence: Enters an infinite execution loop post-initialization to maintain residence in memory.
