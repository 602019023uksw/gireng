1. Executive Summary

The analyzed binary is a statically linked x86_64 ELF executable containing a substantial implementation of the OpenSSL cryptographic library. The primary capabilities observed in the decompiled code include advanced cryptographic transformations (AES, SHA, SEED), SSL/TLS protocol handling (SSLv3, DTLS), and data compression routines.

While the code structure is typical of a legitimate cryptographic library, the presence of embedded strings referencing Google APIs—specifically for OAuth2 (oauth2.googleapis.com) and Google Sheets (sheets.googleapis.com)—along with specific access scopes, suggests the binary is a custom-built tool or client designed to communicate with these services. The binary utilizes static linking to obscure dependencies and includes hardware-optimized crypto routines (AES-NI). Based on the provided code snippets, which consist entirely of standard library implementations for cryptography and networking without explicit malicious payload logic (such as data exfiltration or system exploitation), the binary is assessed as Suspicious. It exhibits the infrastructure for encrypted communication but lacks the visible code to definitively classify it as malware, though the hardcoded Google API strings are atypical for standard library builds and warrant caution.

4. Technical Analysis

In-depth technical examination of the malware code structure, algorithms, and implementation details.
Cryptographic Library Implementation

The binary contains a full suite of cryptographic functions characteristic of the OpenSSL library. This includes low-level assembly implementations of AES (with Intel AES-NI support), SHA-1, SHA-256, SHA-512, and the SEED cipher. The presence of optimizations like "AESNI-CBC+SHA1 stitch" and "gcm_ghash_clmul" indicates a focus on performance.

SSL/TLS Protocol Stack

Functions for SSLv3 and DTLS (Datagram TLS) are present (ssl3_connect, ssl3_accept, dtls1_connect). The code handles session resumption via d2i_SSL_SESSION, parsing ASN.1 encoded session data. This suggests the binary acts as a secure client or server agent.

Embedded API Strings

Unlike standard library builds, this binary includes hardcoded JSON templates and URLs for Google's OAuth2 and Sheets API. This implies the binary is a statically linked utility designed to authenticate and interact with Google services, likely using Sheets as a transport or storage mechanism.

The binary is a statically linked x86_64 executable embedding the OpenSSL library. It contains strings and HTTP templates specifically designed to interact with Google's OAuth2 and Sheets APIs. While the decompiled code provided consists entirely of standard cryptographic library implementations (AES, SHA, SSL) rather than explicit malware payload code, the use of static linking, the presence of hardcoded API endpoints/scopes, and the lack of a clear standard application signature make it suspicious. It likely functions as a custom tool for interacting with Google services, which could be used for legitimate automation or data exfiltration/C2.
