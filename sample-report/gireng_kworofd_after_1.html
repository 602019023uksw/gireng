1. Executive Summary

This binary is a Linux ELF malware (x86-64) that utilizes the Google Sheets API as a Command and Control (C2) infrastructure. By leveraging legitimate cloud services (OAuth2 and Google Sheets), the malware attempts to bypass traditional network detection mechanisms. The binary is statically linked with OpenSSL, indicating it was compiled to be standalone.

The malware operates by decrypting a configuration file (passed as an argument) containing OAuth2 credentials (Service Account keys). It uses these credentials to generate signed JSON Web Tokens (JWTs), exchanges them for an OAuth2 Bearer token, and authenticates to Google APIs. It then enters a loop where it reads commands from a specific Google Spreadsheet cell, parses them, executes them on the local system (including shell commands via popen), and writes the output back to the spreadsheet.

Threat Level: High. The malware demonstrates full C2 capabilities, including system reconnaissance (hostnames, IPs, user info), remote code execution, and data exfiltration, all hidden behind whitelisted Google domains.

4. Technical Analysis

In-depth technical examination of the malware code structure, algorithms, and implementation details.
C2 Communication & Authentication

The malware uses the Google Sheets API v4 for C2. It authenticates using the OAuth 2.0 Server-to-Server flow (JWT Bearer Grant).

  1. Config Loading: fcn.00408190 loads a file path (argv[1]), reads AES-encrypted data, decrypts it, and Base64 decodes it to retrieve the Service Account Email and Private Key.
  2. Token Acquisition: fcn.00408960 constructs a JWT header and payload using the retrieved credentials. It signs the JWT (fcn.004087e0) using the private key. It then sends a POST request to oauth2.googleapis.com/token to exchange the signed JWT for an OAuth2 access_token.
  3. Data Transfer:
    • Read (C2 Check): fcn.00409110 sends a GET request to sheets.googleapis.com to read a specific cell (using valueRenderOption=FORMULA).
    • Write (Exfil/Status): fcn.004093b0 sends a POST request to batchUpdate to write data back to the sheet.
      • Command Protocol

The main function implements a polling loop.

  1. It reads data from the spreadsheet.
  2. It parses the response looking for a command string delimited by dashes (-). Evidence: strchr(rbx, '-').
  3. It uses strcasecmp to match specific keywords in the command parts.
  4. If a match is found, it calls fcn.00407310 to execute the command argument.
    5. System Reconnaissance

fcn.00408b20 acts as a fingerprinting function. It calls gethostname and getifaddrs to identify the machine and its IP addresses. It uses getnameinfo to format IPs. It calls uname for kernel info, getuid/getpwuid for the username, and getcwd for the current path. This data is formatted into a string (likely exfiltrated).

Command Execution

fcn.00407310 creates a shell command string by appending &>2 & to the input command (redirecting stderr and backgrounding). It uses popen to execute it and reads the output stream back into a buffer, which is then returned to the caller.

7. Operational Flow

  1. Initialization: Binary starts. main calls fcn.00408190 with the first argument (path to a config file).
    • Evidence: fcn.00408190 (*((rsi + 8))) @ 0x004095c0
  2. Setup: fcn.00408190 reads the config file, decrypts it using AES, and decodes it to extract the OAuth2 Service Account credentials (Email, Private Key). fcn.00408960 uses these credentials to generate a JWT and retrieves an OAuth2 Access Token from oauth2.googleapis.com.
    • Evidence: AES_cbc_encrypt () @ 0x00408190; rax = fcn_00407940 ("oauth2.googleapis.com"...) @ 0x00408960
  3. Reconnaissance: fcn.00408b20 is called to gather system information (Hostname, IPs, User, OS).
    • Evidence: rax = fcn_00408b20 () @ 0x004095c0
  4. C2 Registration: The recon data is likely formatted and sent to the C2 sheet via fcn.004093b0 (Write).
    • Evidence: fcn.004093b0 (rsp + 0x740, rsp + 0x750) @ 0x004095c0
  5. Command Loop: main enters an infinite while(1) loop.
    • Poll: It calls fcn.00409110 (Read Sheet) to fetch commands.
    • Parse: It checks if the data contains dash-separated keywords.
    • Execute: If a command is found, it calls fcn.00407310 (Exec).
    • Exfil: The result of the execution is sent back via fcn.004093b0 (Write Sheet).
    • Evidence: do { ... rax = fcn_00409300 ... } while (1); @ 0x004095c0
  6. Persistence: No specific persistence mechanism (installing cron/systemd service) is visible in the provided decompiled code; the binary appears to be a standalone payload intended to be executed once (likely by a dropper).

8. C2 & Networking

  • C2 Servers:
  • oauth2.googleapis.com — Purpose: Authentication (OAuth2 Token Exchange). Evidence: String @ 0x00509006 (referenced in fcn.00408960).
  • sheets.googleapis.com — Purpose: Data Transfer (Commands/Results). Evidence: String @ 0x004f4774 (referenced in fcn.00409110).
  • www.googleapis.com — Purpose: API Scopes. Evidence: String in fcn.00408960.
  • Protocols: HTTPS (HTTP/1.1). Evidence: POST ... HTTP/1.1 strings in multiple functions.
  • Communication Pattern: The malware acts as a client interacting with a REST API. It does not use raw TCP sockets directly but constructs HTTP requests.
  • Command Format: Text-based string separated by dashes (-). e.g., COMMAND_TYPE-COMMAND_ID-ARGUMENTS.
  • Authentication: OAuth2 Bearer Token obtained via JWT (JSON Web Token) signed with RS256 (RSA). Evidence: {"alg":"RS256"..." in fcn.00408960.
  • Data Format: JSON (for API payloads) and Plain Text (cell values in Sheets). Base64 for encoding JWT parts.

9. Recommendations

1
Block API Access: If the Service Account credentials used by this malware are known, revoke them immediately in the Google Cloud Console.
2
Network Monitoring: Monitor network traffic for connections to sheets.googleapis.com and oauth2.googleapis.com with the specific User-Agent strings identified (Google-HTTP-Java-Client/1.42.3). While these are legitimate, the specific usage pattern (high frequency polling) might be anomalous.
3
Endpoint Scanning: Scan systems for the configuration file passed as an argument to the binary. It likely contains the stolen OAuth2 private key.
4
Process Monitoring: Monitor for processes executing popen or system calls that originate from unknown binaries, especially those making network requests.
